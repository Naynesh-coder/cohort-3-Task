Browser Rendering Pipeline

This is what happens inside the browser from the moment it receives your HTML and CSS code to the moment you see something on the screen.

1. HTML

The browser downloads the raw HTML file as plain text from the server.

↓

2. Parsing & Tokenization

The browser reads the HTML text and breaks it into small pieces called tokens (like start tags, end tags, and text).

↓

3. DOM Tree

Tokens are turned into objects and connected together to form the DOM (Document Object Model) tree.

↓

4. CSS Parsing

The browser reads all CSS rules from style tags and linked stylesheets.

↓

5. CSSOM Tree

CSS rules are converted into the CSSOM (CSS Object Model), a tree of style information.

↓

6. DOM + CSSOM

The browser combines the DOM and CSSOM together so it knows both structure and style.

↓

7. Render Tree

Only the visible elements (not display:none ones) are kept to build the Render Tree.

↓

8. Paint

The browser fills in pixels - colors, text, images, borders and shadows.

↓

9. Screen Display

The final picture is shown to the user on the screen. This whole process repeats whenever something changes!
